Deploying an Application on ECS EC2 using AWS CodePipeline

This project demonstrates the manual deployment of an application to Amazon ECS (Elastic Container Service) on an EC2 instance using AWS CodePipeline. The setup enables continuous integration and delivery (CI/CD) for automating the build, test, and deployment stages.

Project Overview

  • Infrastructure: Set up an Amazon ECS cluster on EC2 instances.
  • CI/CD Pipeline: Configured AWS CodePipeline to automate application deployment from a GitHub repository.
  • Automation: Utilized CodeBuild and CodeDeploy to manage the deployment stages.

Steps Involved

  1. Set up an ECS Cluster on EC2 instances to run the application.
  2. Configure AWS CodePipeline for continuous integration and deployment.
  3. Use CodeBuild to automatically build and test the application code.
  4. Deploy the application to ECS EC2 instances using CodeDeploy.

Benefits

  • Automated application deployments, reducing manual intervention and deployment times.
  • Improved version control and faster releases by integrating with GitHub.
  • Enhanced scalability and flexibility by deploying on ECS with EC2 instances.
